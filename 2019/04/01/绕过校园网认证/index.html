绕过校园网认证

发表于 2019-04-01 | 更新于 2019-04-03 | 分类于 校园网

　　大概一个月前阿里云活动，通过学生认证的购买服务器有很大的优惠，我就买了三个月的上海的服务器，本想建站呢，无奈备案过程太过复杂繁琐，就扔在一边不管了，这个月校园网到期了，平常又不怎么用，流量又用不完，所以就没续费，突然有个想法，绕过校园网认证上网。
　　我们连接校园网，提示需要登录。

　　我们到终端ping一下ip地址:115.239.210.27，此时是ping不通的，显然这是一个内网，通过这个内网登录到公网，经过认证成功后才能上网。

所以整个认证过程是：

内网登录——公网认证——正常上网

绕过过程是：

内网（不登陆）——绕过学校公网——连接服务器——正常上网

　　大概就是这样

　　那么怎样绕过呢？答案就是依赖端口通过UDP转发。重要的端口有三个，53，67，68。

53端口:53端口为DNS(Domain Name Server,域名服务器)服务器所开放,主要用于域名解析,DNS 服务在NT系统中使用的最为广泛。

67、68端口:67、68端口分别是为Bootp服务的Bootstrap Protocol Server(引导程序协议服务端)和Bootstrap Protocol Client(引导程序协议客户端)开放的端口。

　　SSR只可惜不代理UDP，只代理TCP，所以我们还是需要用OpenVPN，另一款优秀的代理软件。

　　搭建过程就不说了，和SSR搭建过程差不多，这里贴上脚本，按照提示一步步输入并回车就行了。

　　打开juiceSSH，输入port，分别选择1，2，并放行53，68端口，也就是53，68的TCP，DUP都要放行。


　　这里我只截图了放行UDP的，TCP也放行吧，哪个端口能用，取决于学校，一般是53，67，68。我们学校是68，这个自己测试。

　　打开软件，登录你的账号密码（可后台添加和自行注册），选择线路并安装，再链接校园网，点击连接就可以了

　　测一下速度，下载速度0.64M/s，我的服务器只有5Mbps，可以满速，实际使用中网速超过1M/s也是可能的


　　最后放上我用平板测试的截图，B站，未登陆校园网，1080P可以看

以下仅适用于我校，其它院校仅供参考

①Android

下载Android客户端，点我下载注册登录即可，体验时间只有1天和512M流量，可联系我增加时长和流量。点击线路安装，选择校园网并用此应用打开即可。

②IOS

安装OpenVPN客户端，客户端暂不提供，需要美国ID，自行注册或购买，软件名称：OpenVPN

使用Safari打开我，注册或登录，点击右上角菜单，点击线路安装

选择“在OpenVPN打开”

点击ADD

输入账号密码，点击右上角ADD

点击Allow

断开有效WIFI，连接zut-stu，等加载完毕登录页面，点击右上角“取消”，选择“不连接互联网使用”，有可能点击取消直接断开WIFI了，多试几次。

链接校园网后打开OpenVPN，点击按钮连接即可

连接成功后

③Windows

先连接有效网络点我下载Windows客户端，解压到桌面，并运行exe执行文件，一路安装即可。

安装完就可以连接校园网了。

运行软件，右下角软件右键打开菜单，选择“Import file…”

选择刚才解压文件夹里的配置文件

右下角软件右键打开菜单，选择“Connect”，输入用户和密码，

连接成功，软件颜色变成绿色。
